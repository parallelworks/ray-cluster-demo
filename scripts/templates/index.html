Ray Cluster Ready

The multi-site Ray cluster is running with no demo workload. You can submit custom Ray jobs using the instructions below.

1. Connect via SSH Tunnel

From your local machine, open an SSH tunnel to the Ray head node's GCS port (6379): 

ssh -L 6379:<RAY_HEAD_IP>:6379 -o ProxyCommand="pw ssh --proxy-command %h" $USER@<head-resource>
2. Submit a Ray Job (Python)

Connect to the cluster and submit tasks from a local Python script: 

import ray

# Connect to the remote cluster via your SSH tunnel
ray.init(address="ray://127.0.0.1:10001")

# Or connect directly if running on the head node:
# ray.init(address="<RAY_HEAD_IP>:6379")

@ray.remote
def my_task(x):
    import time, socket
    time.sleep(1)
    return f"Result {x} from {socket.gethostname()}"

# Submit tasks across the cluster
futures = [my_task.remote(i) for i in range(10)]
results = ray.get(futures)
for r in results:
    print(r)

ray.shutdown()
3. Submit via Ray Job Submission API

Use the Ray Jobs CLI to submit scripts to the cluster. First tunnel to the Ray dashboard port (8265): 

# Tunnel to the Ray dashboard
ssh -L 8265:<RAY_HEAD_IP>:8265 -o ProxyCommand="pw ssh --proxy-command %h" $USER@<head-resource>

# Submit a job
ray job submit --address http://127.0.0.1:8265 -- python my_script.py

# Check status
ray job list --address http://127.0.0.1:8265
4. Run Directly on the Head Node

SSH to the head node and run scripts directly: 

# SSH to head node
pw ssh <head-resource>

# Activate the Ray environment
source <job_dir>/.venv/bin/activate

# Set the Ray address
export RAY_ADDRESS="<RAY_HEAD_IP>:6379"

# Run your script
python my_script.py
